## [2026-04-27] 事故 G：README 腦補與檔案管理失靈
- **現象**：AI 在 `CONSTITUTION.md` 中規定必須研讀 `README.md`，但實際檢查時發現根目錄並無此檔案。AI 隨後自行建立了一份全新的 `README.md`，導致歷史文檔內容可能遺失或產生斷層。
- **成因**：
    1.  **傲慢假設**：AI 憑直覺認為專案必有 README。
    2.  **查核不實**：在修改憲法時，未先以 `list_dir` 確認檔案存在，導致指令與現實脫鉤。
- **教訓**：在將任何檔案路徑寫入規則（Constitution/Rules）前，必須 100% 執行物理查核。嚴禁在規則中引用「不存在的幻覺檔案」。
- **修正**：已手動重建 `README.md` 並整合目前的法庭交互鏈邏輯。

## [2026-04-27] 事故 F：傲慢抄寫致物理引擎崩潰
- **現象**：新建立的起訴書（IndictmentBook）往回翻頁邏輯完全失效，出現角度穿透、封面消失、無法翻動等 Bug。
- **原因**：AI 助手在搬移「創業卷宗」黃金代碼時，犯下 **「傲慢之罪」**。自以為是地進行了所謂的「精簡」，私自刪除了 Ref-State 雙軌同步補償、GSAP 實時查詢與邊界防穿透判定。
- **後果**：嚴重破壞了手感的一致性，且因擅自刪除重要代碼造成開發進度倒退與開發者信任喪失。
- **戒律**：
    1. 當使用者提供已驗證的基準代碼時，AI 必須 **100% 逐行對齊**，嚴禁任何私自精簡或邏輯壓縮。
    2. 虛心接受人類調教出的物理細節，認清 AI 直覺與真實物理體感的差距。
    3. 執行前必須核對行數與邏輯分支完整性，不得先斬後奏。

## [2026-04-27] 事故 B：iOS 輸入框點擊自動縮放
- **現象**：手機玩家點擊任何 input 時，畫面會突然放大並位移，導致 UI 跑版且無法手動縮回。
- **原因**：iOS Safari 規定字體小於 16px 的輸入框在聚焦時會自動觸發 Accessibility Zoom。
- **解決**：強制所有 input 為 `font-size: 16px` 並搭配 `user-scalable=no` 與 `touch-action: pan-x pan-y`。
