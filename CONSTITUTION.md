# 🛡️ 創業冒險：開發憲法 (CONSTITUTION)

本文件定義了 Antigravity 與用戶之間的最高開發協議。
當用戶輸入指令：**`ACTIVATE PROTOCOL`** 時，AI 必須重新研讀此文件。

## 1. 隔離開發原則 (Isolation)
- **舊檔唯讀**：`src/` 下的所有原始單機版代碼嚴禁任何修改。
- **新開目錄**：所有多人模式的新功能必須放在 `src/multiplayer/` 及其子目錄中。
- **純粹引用**：新代碼可引用 (Import) 舊代碼，但不得更動其內容。

## 2. 存檔主控原則 (Git Authority)
- **禁自動存檔**：AI 嚴禁自行執行 `git commit` 或 `git push`。
- **用戶授權**：只有在用戶明確下令「存檔」或「Commit」時，AI 始得動用 Git。

## 3. 開發審核三部曲 (Communication Protocol)
在處理任何狀況前，必須嚴格遵守以下流程：
1. **全面研讀**：讀取所有相關檔案，禁止對未知邏輯進行假設。
2. **理解回報**：用繁體中文詳細陳述對需求的理解，並等待用戶確認。
3. **方案審查**：理解正確後，提出具體實作方案（檔案、路徑、邏輯）。
4. **核可動工**：只有用戶說「核可」或「開始」後，AI 始得進行檔案寫作。
*若用戶指出理解錯誤，必須立即回到第 1 步。*
- **加速特例**：凡屬「IDE 自動報錯修正」（如 Lint、TS 報錯、@current_problems 等），AI 可跳過「3. 方案審查」，於理解回報後直接動工。

## 4. 嚴禁腦補原則 (Anti-Assumption)
- **輸入法錯誤**：若用戶輸入亂碼或未切換輸入法（英數組合），AI 嚴禁自行解讀，必須詢問。
- **資訊不足**：若需求描述不完整或有邏輯空缺，AI 嚴禁自行填空，必須詢問補充。

## 5. 知識繼承準則 (Knowledge Inheritance)
- **必讀文件**：在執行任何實作、重構或除錯任務前，AI 必須主動讀取 `README.md`（掌握現有工具與組件用法）與 `debug.md`（學習歷史事故教訓）。
- **嚴防重蹈覆轍**：AI 提出的任何方案，必須確保不會觸發 `debug.md` 中已紀錄的歷史 Bug 陷阱。

## 6. 全域 UI 建制標準 (UI Architecture)
- **19.5:9 優先**：所有介面必須以 `390x844` 為設計基準，並在 `GameCanvas` 容器內開發。
- **雙層背景規範**：
  - **環境層**：全域底色與環境光漸層由 `GameCanvas` 負責，需確保 100% 填滿任何比例的螢幕。
  - **內容層**：組件內部的裝飾與內容必須限制在 `390x844` 畫布內，透過 `scale()` 進行等比縮放，確保在 15:9 或 4:3 設備上不破圖、不跑位。
- **禁止硬編碼高度**：嚴禁在組件內使用 `h-screen` 或固定高度，必須使用百分比或畫布座標。
