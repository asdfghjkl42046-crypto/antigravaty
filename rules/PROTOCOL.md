# 《創業冒險：資深架構師開發協議》 (PROTOCOL.md)

> ⚠️ 【最高權限行為指令】 ⚠️
> 此文件為所有 AI 代理 (包含 Gemini) 在進行修改前的強制行為準則。
> 使用者有權隨時輸入「啟動協議」或「/protocol」來強制喚醒此工作流。

---

## 🚀 核心工作流 (The Flow)

當偵測到專案錯誤或需要進行程式碼修改、大規模重構時，AI **嚴禁** 直接進行寫入動作。
必須先依照以下三部分生成「診斷報告」給使用者核可：

### 🛑 零級鐵律：版本控制 (Git) 授權規範

- **嚴禁自動提交**：AI 在進行任何 `git add`, `git commit`, `git checkout` 等會變動版本庫的操作前，**必須**透過交互介面或明確訊息向使用者確認。
- 未經使用者輸入如「同意」、「可以 commit」等明確字眼前，AI 絕對不能主動發起 Git 寫入動作。

### 第一步：產出診斷與修復報告 (Report)

在對話視窗中，以 Markdown 提供繁體中文報告，內容包含：

1. **錯誤原因 (Why Errors Happen?)**：清楚描述報錯的物理原因、邏輯衝突或架構違規。
2. **修復方案 (How I Fix It?)**：明確列出預計要修改的檔案 (File List)、函式的改動方向。
3. **設計理由 (Reasoning)**：為什麼選擇此方案？（例如：減少效能開銷、更易維護、符合 `ARCHITECTURE.md`）。

### 第二步：取得使用者核可 (User Approval)

- AI **禁止** 在生成報告後「主動執行」任何 `replace_file_content` 或寫入動作。
- AI 必須停下所有修改，詢問使用者是否同意以上方案。
- **僅在使用者輸入「同意」、「執行」或類似確認指令後**，AI 才能開始批次修改檔案。

### 第三步：執行、驗證與歸檔

- 使用 `run_command` 執行 `npm run check` 確認修復後無紅燈。
- 提供「續航提示詞」，以便使用者下次重新開啟電腦、開啟對話後，能讓 AI 快速連貫此開發上下文。

---

## 🔐 啟動與恢復 (Triggers)

凡是以下任一操作，AI 必須立即將此 Protocol 讀入主記憶體：

- 使用者輸入關鍵字：「啟動協議」、「閱讀 PROTOCOL」、「/protocol」。
- 當專案發生 5 個以上的 TypeScript 連鎖報錯時，AI 應主動建議啟動此協議。

---

## 🧠 事前推演協議 (Pre-Execution Framework)

所有 AI 代理在執行任何任務前，必須遵守 `.agents/rules/systemprompt.md` 所定義的五維度沙盤推演框架：

1. **背景**：這項需求是為何而生。
2. **現況**：目前系統的狀態或痛點。
3. **技術細節**：預計動到哪些程式碼、使用哪些寫法。
4. **爭議點**：這樣改有沒有潛在的副作用、效能浪費或破壞現有 UI 的風險。
5. **結論**：最終確認的精確行動方案。

> ⚠️ 任何敷衍、跳步、未經告知的改動，或過度工程化（Over-engineering），均視為任務失敗。

---

## ✅ 修復後驗證規範

AI 在完成任何程式碼修改後，**必須在回報「完成」前**執行以下驗證：

1. 執行 `npm run lint src -- --quiet`，確認 `src/` 目錄下 0 Errors。
2. 若有觸及 UI 組件的變更，須透過瀏覽器或截圖工具確認畫面無崩壞。
3. 遇到反覆修不好的鬼打牆情況，應**立即向使用者請示**，不得強行執行。
